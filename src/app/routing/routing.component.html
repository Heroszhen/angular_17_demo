app.routes.ts

Définir les routes dans ce fichier

Dynamique et paresseux

component: HomeComponent // Chargement immédiat
loadComponent: () => import('./routing/routing.component').then(m => m.RoutingComponent) // Chargement paresseux d'un composant

Navigation

Dans une balise a :
routerLink="news" routerLinkActive="active"
routerLinkActiveOptions(entre les crochets) = "exact: true(entre les accolades)"

ActivatedRoute: url, params, queryParams(récupérer)
Router: navigate (naviguer)
Router: events(router listener dans app.component.ts)

Les Guards

canActivate
ng generate guard auth (implements CanActivate dans la classe)

Dans app.routes.ts:
path: 'dashboard',
component: DashboardComponent,
canActivate: [AuthGuard]
canLoad
ng generate guard auth2 (implements CanLoad dans la classe)

Dans app.routes.ts:
path: 'articles',
component: ArticlesComponent,
canLoad: [Auth2Guard]
canMatch(A partir de angular 15)
ng generate guard auth3 (implements CanMatch dans la classe)

Dans app.routes.ts:
path: 'produits',
component: ProduisComponent,
canMatch: [Auth3Guard]
canActivate : Quand tu veux protéger une route après qu'elle a été trouvée(sélectionnée) dans la configuration du routage, mais avant de l'activer. Authentifié

canMatch : Quand tu veux bloquer l’accès à une route avant même qu’elle soit matchée ou chargée (sélectionnée). C'est utile pour des vérifications très préventives. En fonction des permissions

canLoad : Quand tu veux éviter le chargement du module lazy-loaded pour certaines routes, basé sur la condition que l'utilisateur doit remplir avant que le module soit téléchargé.
